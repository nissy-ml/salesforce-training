# Day 2 - Salesforce Platform Basics

## 1. What is Salesforce Platform?
Salesforce Platform is a cloud-based environment that allows businesses to build, customize, and manage applications for CRM, automation, and customer engagement.

It provides tools for both configuration and development, enabling businesses to create scalable solutions without managing infrastructure.

---

## 2. Core Components of Salesforce Platform

### App
An App is a collection of tools, tabs, and features designed for specific business functions.

Example:
- Sales App
- Service App

---

### Object
An Object is a database table used to store specific business data.

Examples:
- Account
- Contact
- Opportunity

Objects contain records and fields.

---

### Tab
A Tab is the user interface that allows users to access objects, apps, or custom pages easily.

Example:
- Accounts Tab
- Contacts Tab

---

## 3. Configuration vs Coding

### Configuration
Using built-in Salesforce tools like:
- Flow
- Process Builder
- Validation Rules

Best when:
- No-code/low-code solution is enough
- Faster implementation needed

---

### Coding
Using:
- Apex
- Visualforce
- Lightning Web Components

Best when:
- Complex business logic
- Advanced customization
- External integrations

## 5. Multi-Tenant Architecture (Basic Understanding)
Salesforce uses multi-tenant architecture, meaning multiple customers share the same platform infrastructure while keeping their data secure and separate.

This makes Salesforce scalable, cost-effective, and easy to maintain.

---

## 6. How Salesforce Extends Functionality
Developers can extend Salesforce using:

- APIs
- AppExchange integrations
